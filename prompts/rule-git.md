Code changes happen ONLY inside a git worktree. Branches are for remote
history; locally, every branch you work on lives in its own worktree with

## Template: `.claude/hooks/enforce-git-naming.sh`

A `PreToolUse` hook on `Bash` that enforces naming conventions for four git operations.
Copy this template verbatim — the sourced `git-conventions.sh` provides all project-specific
values so the hook itself needs no editing when conventions change.

```bash
#!/usr/bin/env bash
# PreToolUse enforcement: validate git operation naming conventions.
# Generated by claude-harness rule-git task.
# Exit 0 = allow. Exit 2 = block (stderr becomes Claude's feedback context).
#
# Enforces:
#   git checkout -b        → blocked (use git worktree instead)
#   git worktree add -b    → branch name must match BRANCH_PATTERN
#   git push <protected>   → direct push to protected branches blocked
#   git commit -m          → message must match COMMIT_PATTERN

set -euo pipefail

[ "${CLAUDE_HARNESS_SETUP:-}" = "1" ] && exit 0

INPUT=$(cat)
COMMAND=$(echo "$INPUT" | python3 -c "import json,sys; d=json.load(sys.stdin); print(d.get('tool_input',{}).get('command',''))" 2>/dev/null || echo "")
CWD=$(echo "$INPUT" | python3 -c "import json,sys; d=json.load(sys.stdin); print(d.get('cwd',''))" 2>/dev/null || echo "$(pwd)")

# Fast path: only act on git commands we enforce.
case "$COMMAND" in
  *"git checkout"*"-b"*|*"git checkout"*"--branch"*|\
  *"git worktree add"*|\
  *"git commit"*"-m"*|\
  *"git push"*) ;;
  *) exit 0 ;;
esac

# Source naming conventions. No file = no enforcement.
GIT_TOP=$(git -C "$CWD" rev-parse --show-toplevel 2>/dev/null || echo "")
CONVENTIONS="${GIT_TOP}/.claude/git-conventions.sh"
[ ! -f "$CONVENTIONS" ] && exit 0
# shellcheck source=/dev/null
source "$CONVENTIONS"

# ── git checkout -b : always block (worktree-first) ──────────────────────
if echo "$COMMAND" | grep -qE 'git checkout .*(--branch|-b)'; then
  echo "Blocked: 'git checkout -b' creates a branch in the main working tree." >&2
  echo "This repo requires worktree-first development." >&2
  echo "" >&2
  echo "Open a worktree instead:" >&2
  echo "  git worktree add ../<dir> -b <branch>" >&2
  echo "" >&2
  echo "Branch naming standard: ${GIT_WORKFLOW_DOC:-.claude/rules/git-workflow.md}" >&2
  exit 2
fi

# ── git worktree add -b <branch> : validate branch name ──────────────────
if echo "$COMMAND" | grep -qE 'git worktree add '; then
  BRANCH=$(echo "$COMMAND" | sed -nE 's/.*(-b|--branch)[[:space:]]+([^[:space:]]+).*/\2/p')
  if [ -n "$BRANCH" ] && [ -n "${BRANCH_PATTERN:-}" ]; then
    if ! echo "$BRANCH" | grep -qE "$BRANCH_PATTERN"; then
      echo "Blocked: branch name '$BRANCH' doesn't match the project convention." >&2
      echo "" >&2
      echo "Pattern:  $BRANCH_PATTERN" >&2
      [ -n "${BRANCH_EXAMPLES:-}" ] && echo "Examples: $BRANCH_EXAMPLES" >&2
      echo "" >&2
      echo "See ${GIT_WORKFLOW_DOC:-.claude/rules/git-workflow.md} for the naming standard." >&2
      exit 2
    fi
  fi
  exit 0
fi

# ── git push <remote> <protected> : block direct pushes ──────────────────
if echo "$COMMAND" | grep -qE '^git push '; then
  for PROTECTED in ${PROTECTED_BRANCHES:-}; do
    if echo "$COMMAND" | grep -qE "(^|[[:space:]])${PROTECTED}([[:space:]]|$)"; then
      echo "Blocked: direct push to '${PROTECTED}' is not allowed." >&2
      echo "Open a pull request instead." >&2
      echo "See ${GIT_WORKFLOW_DOC:-.claude/rules/git-workflow.md} for the PR process." >&2
      exit 2
    fi
  done
  exit 0
fi

# ── git commit -m <message> : validate commit message ────────────────────
if echo "$COMMAND" | grep -qE 'git commit .*-m'; then
  # Skip heredoc commits — can't reliably parse $(cat <<'EOF' ... EOF)
  echo "$COMMAND" | grep -qE "<<|'EOF'|\"EOF\"" && exit 0

  # Extract -m argument with shlex for reliable quote handling
  MSG=$(echo "$COMMAND" | python3 -c "
import sys, shlex
try:
    parts = shlex.split(sys.stdin.read().strip())
    for i, p in enumerate(parts):
        if p == '-m' and i + 1 < len(parts):
            print(parts[i + 1])
            break
except Exception:
    pass
" 2>/dev/null || echo "")

  if [ -n "$MSG" ] && [ -n "${COMMIT_PATTERN:-}" ]; then
    if ! echo "$MSG" | grep -qE "$COMMIT_PATTERN"; then
      echo "Blocked: commit message doesn't match the project convention." >&2
      echo "" >&2
      echo "Message:  $MSG" >&2
      echo "Pattern:  $COMMIT_PATTERN" >&2
      [ -n "${COMMIT_EXAMPLES:-}" ] && echo "Examples: $COMMIT_EXAMPLES" >&2
      echo "" >&2
      echo "See ${GIT_WORKFLOW_DOC:-.claude/rules/git-workflow.md} for the correct format." >&2
      exit 2
    fi
  fi
  exit 0
fi

exit 0
```

## Template: `.claude/hooks/enforce-worktree.sh`

A `PreToolUse` hook on `Edit|Write` that blocks code edits when Claude is running in the
main git working tree instead of a linked worktree. Sources `git-conventions.sh` so its
error messages show the same branch examples and doc reference as `enforce-git-naming.sh`.

**Detection:** `$GIT_TOPLEVEL/.git` is a *directory* in the main working tree, a *file*
(gitdir pointer) in a linked worktree. One `[ -d ]` check is sufficient and reliable.

```bash
#!/usr/bin/env bash
# PreToolUse enforcement: require a git worktree for all code edits.
# Generated by claude-harness rule-git task.
# Exit 0 = allow. Exit 2 = block (stderr becomes Claude's feedback context).

set -euo pipefail

[ "${CLAUDE_HARNESS_SETUP:-}" = "1" ] && exit 0

INPUT=$(cat)
CWD=$(echo "$INPUT" | python3 -c "import json,sys; d=json.load(sys.stdin); print(d.get('cwd',''))" 2>/dev/null || echo "")
[ -z "$CWD" ] && CWD=$(pwd)

GIT_TOP=$(git -C "$CWD" rev-parse --show-toplevel 2>/dev/null || echo "")
[ -z "$GIT_TOP" ] && exit 0

# Source naming conventions — provides WORKTREE_DOC, BRANCH_EXAMPLES, GIT_WORKFLOW_DOC.
CONVENTIONS="${GIT_TOP}/.claude/git-conventions.sh"
# shellcheck source=/dev/null
[ -f "$CONVENTIONS" ] && source "$CONVENTIONS"

# .git is a DIRECTORY in the main working tree; a FILE in a linked worktree.
if [ -d "$GIT_TOP/.git" ]; then
  BRANCH=$(git -C "$CWD" rev-parse --abbrev-ref HEAD 2>/dev/null || echo "unknown")
  echo "Blocked: on branch '$BRANCH' in the main working tree — not a git worktree." >&2
  echo "This repo requires worktree-first development — no exceptions." >&2
  echo "" >&2
  echo "Open a worktree:" >&2
  echo "  git worktree add ../<name> -b <branch>" >&2
  [ -n "${BRANCH_EXAMPLES:-}" ] && echo "  Valid branch names: ${BRANCH_EXAMPLES}" >&2
  echo "" >&2
  echo "See ${WORKTREE_DOC:-docs/WORKTREE.md} for service provisioning and the full workflow." >&2
  exit 2
fi

exit 0
```

## Rules

- `.claude/rules/git-workflow.md` has NO frontmatter — it loads every session.
- All conventions must come from actual git history — not assumed defaults.
- `.claude/git-conventions.sh` must use the SAME discovered values as `git-workflow.md`.
  They are two representations of the same truth: one machine-readable, one human-readable.
- If this is a greenfield repo (check: `git log --oneline | wc -l` returns a small number,
  or no `.git` directory), write recommended defaults and mark both files as "proposed".
- If any of these files already exist, read them first and MERGE.
- `enforce-git-naming.sh` must be generated unconditionally — even greenfield repos get it.
  It will just enforce whatever defaults are in `git-conventions.sh`.
- The hook sources `git-conventions.sh` at runtime — so updating conventions only requires
  re-generating `git-conventions.sh`, not the hook script itself.
